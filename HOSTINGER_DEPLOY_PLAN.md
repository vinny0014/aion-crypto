# AION Crypto — Hostinger preview and production plan

Prepared: 2026-07-22
Scope: configuration and safe preview only
Hard stop: **do not attach `aioncrypto.cloud`, edit DNS, enable indexing, or merge the PR**

## Target topology

```text
GitHub (source of truth)
├── Hostinger Node.js Web App: Next.js frontend
├── Render Web Service: FastAPI API
└── CI: configuration, frontend, backend, security, E2E

Supabase
├── PostgreSQL
├── future Auth/Storage integration
└── future users, watchlists and articles
```

The backend remains Python. It is not rewritten to Node and is not placed in the Hostinger frontend runtime.

## Current deployment assets

- Frontend standalone output is configured in `frontend/next.config.mjs`.
- `frontend/scripts/prepare-standalone.mjs` copies static assets into the standalone server bundle.
- Node compatibility is declared as `>=22 <25`; `.nvmrc` selects Node 22.
- Frontend health endpoint: `GET /health`.
- Render Blueprint: `render.yaml`.
- Backend readiness endpoint: `GET /health/ready`.
- Alembic runs as the Render pre-deploy command.
- Indexing is disabled by default.

Hostinger currently documents Next.js support, Node 18/20/22/24, GitHub-triggered builds, and a temporary domain after deployment. The temporary domain is the required preview surface; the custom domain step is intentionally excluded. See [Hostinger Node.js Web Apps](https://www.hostinger.com/support/how-to-deploy-a-nodejs-website-in-hostinger/) and [Hostinger deployment workflow](https://www.hostinger.com/tutorials/deploy-node-js-application).

## Gate 1 — Supabase preview project

Create a dedicated non-production Supabase project or branch. Do not reuse AION News or any unrelated database.

1. Obtain the session-pooler URL from the Supabase **Connect** dialog. A long-lived Render process should use direct connectivity where available; if Render needs IPv4 pooling, use session mode, not transaction mode.
2. Convert the SQLAlchemy scheme to `postgresql+psycopg://` without changing the remaining credential fields.
3. Store the URL only as Render `DATABASE_URL`.
4. Run `alembic -c alembic.ini upgrade head` through the configured pre-deploy step.
5. Confirm `/health/ready` returns 200.
6. Create the first admin interactively with `scripts/seed_admin.py`; do not log the password.
7. Verify backup/restore for the chosen Supabase plan before production approval.

Supabase documents connection modes and the limitation of transaction pooling in [Connect to Postgres](https://supabase.com/docs/guides/database/connecting-to-postgres), and migration discipline in [Database Migrations](https://supabase.com/docs/guides/deployment/database-migrations).

## Gate 2 — Render API preview

Create a Blueprint deployment from `render.yaml` on the review branch.

| Setting | Value |
|---|---|
| Type | Python web service |
| Root | `backend` |
| Build | `pip install --upgrade pip && pip install -r requirements.txt` |
| Pre-deploy | `alembic -c alembic.ini upgrade head` |
| Start | `uvicorn app.main:app --host 0.0.0.0 --port $PORT --proxy-headers` |
| Health | `/health/ready` |
| Preview | Manual |
| Auto-deploy | Only after checks pass |

Set `FRONTEND_URL` to the eventual Hostinger temporary HTTPS origin and place any additional temporary origin in `CORS_ORIGINS`. `DATABASE_URL` is the Supabase preview connection. `JWT_SECRET` is generated by Render. Keep `PUBLIC_SITE_URL=https://aioncrypto.cloud` as the canonical identity.

Render's official FastAPI quickstart uses Uvicorn with `$PORT`, and the Blueprint reference supports `rootDir`, pre-deploy migrations, health checks, CI-gated deploys and manual previews: [FastAPI on Render](https://render.com/docs/deploy-fastapi), [Blueprint YAML reference](https://render.com/docs/blueprint-spec).

Do not provision scheduler or agent workers yet. There is no production scheduler command or registered handler set in this repository. Provisioning empty workers would create cost and a false operational signal.

## Rollback

Frontend:

1. Record the last approved Git commit and Hostinger deployment ID before every redeploy.
2. On failure, redeploy the previous approved commit/branch state from hPanel.
3. Keep indexing disabled throughout preview rollback.

Backend:

1. Roll Render back to the previous healthy image/commit.
2. Treat database migrations as forward-compatible. Never assume an application rollback reverses schema safely.
3. Restore Supabase only from a verified backup and only with explicit data-owner approval.

Release:

- Merge remains a separate human action after review.
- DNS change remains a separate human action after production acceptance.
- There is no automatic DNS or custom-domain step in this plan.
